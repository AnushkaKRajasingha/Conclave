Configuration files

An overview of the important configuration settings for when you need to change your basic setup.

Autoload

application/config/autoload.php

We are autoloading these 3 on LINE 63:
$autoload['libraries'] = array('database', 'session', 'template');

We autoload out settings model at the bottom of the file:
$autoload['model'] = array('utils/settings_model');

Config

application/config/config.php

$config['base_url'] on LINE 21: is set dynamically so when the application is moved this doesn't need to be changed.

The following are set automatically by the installer:

$config['encryption_key']
on LINE 309 - this key is used by the encryption class. We use this key to encrypt and decrypt email data.
$config['sess_cookie_name']
on LINE 360 - must be a unique value for example when running multiple instances of this application.
$config['cookie_domain']
on LINE 383 - by default is set to site-wide cookies on your domain - not needed for localhost setups.

Extra feature: turning off CSRF regeneration.
LINE 437-446 - add controller/method to the $no_regen array if you want to turn of regeneration for this cookie. This is sometimes needed for JavaScript-heavy plugins where building in the CSRF regeneration isn't recommended. The problem with AJAX and CSRF is that when working with multiple tabs on the same application the token will differ which causes errors. One solution would be to disable CSRF for those methods but this way you can just disable the regeneration and the tokens will continue to work.

Constants

application/config/constants.php

The two custom CI Membership settings here are set by the installer:

  • The ADMINISTRATOR global constant is used for the root admin account, the only admin account that can never be deleted.
  • SITE_KEY is currently used as a secret salt to help create the renew_password token.

Database

application/config/database.php

There is nothing out of the ordinary here, our installer sets all the values. You can set PDO by adding a dsn and changing dbdriver to pdo.

The constant DB_PREFIX is added for CI Membership and should be used in all queries for example:

$this->db->from(DB_PREFIX .'user u');

Routes

application/config/routes.php

There are a few custom routes to shorten the auth pages.

Default controller = home:

$route['default_controller'] = 'home';

.htaccess file

Takes care of removing the index.php from the url and has some examples for redirecting to https and removing www: both these operations will improve SEO and usability.
For Nginx you need a separate file, for a great example please visit https://www.nginx.com/resources/wiki/start/topics/recipes/codeigniter/.
