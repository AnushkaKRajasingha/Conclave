Core Modules

This application uses HMVC from Wiredesignz for a more modular approach to our coding needs. This means that all our code resides inside application/modules and some sort of separation becomes possible. Adding new functionality means adding a new module with the same folder structure.
Unfortunately we still can't include our assets (images, css, js) in these module folders because this would mean an insecurity so if you want to separate your assets you can create separate folders in assets. For this application however I opted for a vendor folder instead: assets are probably shared between modules at some point and keeping them in their vendor-specific folder allows for easy referencing. You can still use the regular MVC design pattern CodeIgniter uses, it's just that HMVC has many advantages over regular MVC in CodeIgniter that it is highly advised to use it.

adminpanel

The backend module. All core management functionality has been collected into this module: - site settings - admin dashboard - member management - OAuth provider setup - roles and permissions

auth

Takes care of all authentication aspects of the application be it social or regular login and registration.

  • login
  • registration
  • social login and signup
  • authentication recovery in case of lost data

home

The default public page needs a controller just like any other page. The home controller has been designated as default controller for all modules. There are no models involved in this module, just a simple controller. This can easily be used as a base to start from.

membership

For all the pages that are related to the member and their data. Currently we have:

  • dashboard
  • profile

In the future a public profile page will also be added and this is just one example of what you can expect in new releases.

utils

Contains various code that doesn't really need it's own module.

controllers:

  • install
  • page not found
  • site offline

modules:

  • RBAC for access control (see roles and permissions)
  • set cookies model (loads data when unique_token cookie is found for remembering logins)
  • settings (autoloaded, writes config to file)

page_layouts

Just a generic folder containing the different controllers for our set of adminpanel layouts. We have six controllers, one for each layout option. Can be removed as they are not actively used anywhere, these are just examples for how to quickly set up a differing layout.
