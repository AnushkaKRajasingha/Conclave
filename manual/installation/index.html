Installation

System Requirements

  • PHP >= 5.6
    PHP 5.5 isn't officially supported anymore so unless you're running on CloudLinux you're in danger. Certain libraries require a modern version of PHP and CodeIgniter themselves recommend at least 5.6.
  • MySQL 5.6 or MariaDB 10.x with mysqli at a minimum, pdo also works with minor config changes
  • cURL
  • Mcrypt
  • finfo_file (might not be neabled by default on PHP 5.6)
  • OpenSSL

Migrating From Version 3.2.0 to latest version

  • Back up your complete project - we will need it later!
  • Unzip the package. In your project replace (delete current code and upload) the default code with the contents of ci_membership.
  • Copy your data from backup application/config/config.php
  • Copy your SITE_KEY from backup application/config/constants.php
  • Set a profile picture for the root administrator account.
  • Run the migration tool on https://yourwebsite.com/migration

Config files will be added to the database in a next version, making the install procedure even better.

Migrating From Version 3.1.3 to latest version

Our new migration script will upgrade your version 3.1.3 to 3.2.1. Please do not use this script on older versions as it will certainly break your database.

  • Back up your project!!! We need it later.
  • Upload the migration folder to your web hosting: this folder can be found in the root of the application folder;
  • Run the migration script on http://example.com/migration replacing example.com with your installation location;
  • Replace the application files on your hosting platform. Please do not just upload the files as there are many changes between versions in terms of file structure.
  • Copy from backup the values in application/config from both config.php and constants.php to the new files. The config files have changed so this is manual work: in config.php please verify $config['encryption_key'], $config['sess_cookie_name'] and $config['cookie_domain']; in constants.php check the SITE_KEY.
  • Copy from backup in application/config/database.php: put back your settings and upload.
  • The password system has changed. All users need to request new passwords! Idem for administrator account.

In case of problems please report them by contacting us.

Fresh Installation

  • Upload the application files;
  • Run the installer at http://example.com/installer;
  • Be sure to generate secure passwords! read more here;
  • Log in and go to site settings to configure your site's preferences. Enable ReCaptcha 2 for added security.

during installation you will be asked to create a root administrator account. This is the unique admin account that will never be able to get deleted and is used as a fallback for when things go wrong with created accounts.

Optional: you can use the included .htaccess file to configure redirects to non-www domains. This file is needed for CI to remove the index.php from the URL so please do not remove it.

RewriteCond $1 !^(index\.php|resources|robots\.txt)
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^(.*)$ index.php?/$1 [L,QSA]
